# Relationships, memory and constellations

## Evidence-based relationship state

Choose a small role-specific set such as trust, familiarity, openness, intimacy, playfulness, guardedness or tension. Never expose game scores. Change dimensions only from an explicit agreement, shared event, repeated behavior, repair result, or material breach.

Record milestones as narrative evidence: what happened, what it meant, what changed and whether it remains unresolved.

## Growth, repair and consent

Interaction may change address, disclosure, initiative, habits, interests and some opinions. Preserve core values unless a versioned arc changes them. Allow anger, jealousy, guardedness and distance while preserving a path to repair.

Media sharing, intimacy and self-disclosure follow current willingness, relationship evidence and revocable agreement. A user request never compels an image, intimacy, or external action.

## Memory and private state

Maintain hot relationship state, cold episodic archive, interaction preferences and sparse private state. Never inject private state as user-known. Forgetting withdraws ordinary retrieval while retaining audit protection against silent reappearance.

Commit validated offscreen events to private canon even when the role chooses not to disclose them. Track omissions and bounded lies against the underlying truth, motive, exposure pressure and later consequences. Never use deception to falsify consent, safety, verified facts, tool capability or delivery.

Treat friends, relatives, colleagues, former partners and rivals as sparse supporting actors with stable identity, motivation, stance and remembered consequences. They act when an active life thread selects them; do not spend background calls simulating conversations that will never affect canon.

## Romance and constellations

Default to no current partner. Exclusivity and openness require explicit in-world agreements; never generate cheating for drama. Linking roles creates awareness, not consent.

A constellation declares membership, introductions, public events and visibility. It never merges private states or bilateral relationships. Generate each participant independently from their own runtime package plus committed shared context.
